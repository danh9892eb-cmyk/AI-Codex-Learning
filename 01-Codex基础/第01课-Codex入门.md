# 第 01 课：Codex 入门

## 本课目标

- 了解 Codex 能帮助我们完成什么。
- 学会用清晰的方式描述任务。
- 掌握“检查、执行、验证”的基本协作流程。
- 知道如何保护已有文件和重要内容。

## 一、Codex 是什么

Codex 是一个可以与你协作完成数字化任务的 AI 助手。它不仅能回答问题，还可以在获得允许后检查项目文件、创建或修改内容、运行检查，并汇报结果。

对于非程序员，可以把 Codex 理解为一位能够操作项目文件、解释技术概念并陪你逐步实践的学习伙伴。

## 二、Codex 可以做什么

- 创建和整理 Markdown 学习笔记。
- 检查文件夹结构，批量建立规范目录。
- 帮助理解 GitHub、Obsidian 和自动化工具。
- 编写或修改代码，并解释每一步的作用。
- 检查任务是否完成，发现遗漏并提出下一步建议。

## 三、怎样描述一个好任务

一个清晰的任务通常包括以下内容：

1. **背景**：你是谁，正在做什么。
2. **目标**：最终希望得到什么结果。
3. **任务清单**：需要完成哪些具体事项。
4. **限制条件**：哪些内容不能删除或修改。
5. **验收标准**：怎样判断任务已经完成。

示例：

> 请先检查当前文件夹，然后创建学习目录和 Markdown 文件。不要删除已有内容。完成后列出新建文件，并告诉我如何验证。

## 四、推荐协作流程

### 1. 检查

先让 Codex 查看现状，避免覆盖已有文件或重复创建内容。

### 2. 执行

让 Codex 按任务清单逐项处理。较大的任务可以拆分成多个小步骤。

### 3. 验证

检查目录、文件名和文件内容是否符合要求。必要时运行测试或人工打开文件确认。

### 4. 记录

把学到的知识、遇到的问题和解决方法写入学习笔记或作业记录。

## 五、安全习惯

- 明确说明“不要删除现有文件”。
- 修改重要内容前，先查看文件或保留版本记录。
- 对不理解的命令，先询问用途和影响。
- 完成后检查实际文件，而不只看文字说明。
- 使用 GitHub 或其他版本管理工具保留修改历史。

## 本课小结

与 Codex 高效协作的关键不是一次写出完美指令，而是给出清晰目标、明确边界，并通过检查和验证逐步完成任务。
