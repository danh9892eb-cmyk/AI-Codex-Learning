# AI-Codex-Learning

这是一个面向非程序员的系统化学习项目，用于记录 Codex、GitHub、Obsidian 和 AI 自动化工具的学习过程，并逐步完成可复用的实战项目。

## 学习目标

- 掌握 Codex 的基本使用方法，能够清晰地描述任务、检查结果并持续改进。
- 理解 GitHub 的核心概念，学会保存、管理和同步自己的学习项目。
- 使用 Obsidian 建立结构清晰、便于检索和复习的个人知识库。
- 了解常见 AI 自动化工具，并能够设计简单、可靠的自动化流程。
- 通过实战项目整合所学内容，形成自己的操作手册和作品集。

## 学习目录说明

| 目录 | 用途 |
| --- | --- |
| `01-Codex基础` | 学习 Codex 的基础概念、操作方法、提示词和任务协作流程。 |
| `02-GitHub学习` | 学习仓库、提交、分支、同步和版本管理。 |
| `03-Obsidian知识库` | 整理 Obsidian 使用方法、知识卡片和双向链接实践。 |
| `04-自动化工具` | 学习 AI 自动化工具、工作流设计和常用案例。 |
| `05-实战项目` | 保存综合练习、阶段项目及成果说明。 |
| `99-课程总结` | 汇总阶段复盘、经验清单和课程总结。 |

## 每日学习规则

1. 每天完成一个小而明确的学习任务，建议用时 30～60 分钟。
2. 学习前写下当天目标，学习后记录结果和遇到的问题。
3. 新知识用自己的语言总结，并至少完成一次实际操作。
4. 不确定的操作先让 Codex 解释，再执行并检查结果。
5. 每次修改后及时保存；学习结束时更新进度和下一步任务。
6. 每周进行一次复盘，将重要经验整理到课程总结中。
